## Vision

### 10x Check

不只是 "Claude Code 使用追踪器"，而是 **AI 开发者身份平台** — "Strava for AI Developers"。

每个开发者有一个公开的 AI 使用画像页，展示他们的 AI 使用统计、模型偏好、使用热力图、成就徽章。像 GitHub 的绿色方块展示你写了多少代码，这个平台展示你用了多少 AI。

核心增长飞轮：CLI 收集数据 → 看板展示 → 卡片分享 → 新用户发现 → 安装 CLI → 循环

### Platonic Ideal

安装一个 CLI，然后忘掉它。它在后台静默收集 AI 使用数据，每周自动生成一张精美的使用卡片。卡片设计好到分享它感觉像是在秀，而不是在打广告。有一个排行榜，AI 重度用户成为开发者社区里的小名人。公司能看到团队的 AI 采用率指标。

## Architecture Decisions

| 决策 | 选择 | 理由 |
|------|------|------|
| 数据输入 | CLI wrapper (npx) → ccusage daily + blocks → 发送到服务端 API | 一条命令，零摩擦 |
| 数据粒度 | daily 聚合（看板/热力图/ROI）+ blocks 时间分布（时间徽章） | ccusage CLI 两个命令组合 |
| 数据处理 | 本地预处理，只上传统计摘要（token数/模型/时间戳/费用/活跃时间分布） | 隐私优先 |
| 卡片生成 | 服务端预生成 OG 图片 → 缓存到 Supabase Storage | 分享链接直接返回静态图 |
| 技术栈 | Next.js + Vercel + Supabase | 全家桶，最快上线 |
| 数据库 | 2 张表: profiles + daily_usage (含 active_hours JSONB) | 简化架构 |
| 渐进增强 | V1: ccusage CLI → V2: 直接日志解析 | 先快后优 |

### Navigation

顶部导航栏（非侧边栏）：`Logo | Dashboard | Profile | ⚙️`
- 登录用户：显示完整导航
- 访客：Logo + 登录/注册 CTA
- 移动端：Logo + 汉堡菜单

### Dashboard 信息层级

1. **一级（5秒）**：概览卡片行 — 总 Token、等价 API 费用、ROI 节省、活跃天数
2. **二级（30秒）**：使用趋势折线图 + 模型分布图 + 时间范围选择器
3. **三级（探索）**：热力图 + 最近成就 + 分享卡片预览/生成 CTA

### Profile 信息层级

1. **一级**：头像 + 用户名 + 核心三数字（总token/总费用/活跃天）
2. **二级**：热力图（最视觉化元素）
3. **三级**：徽章墙 + 历史卡片

## Interaction States (Design Review)

| 功能 | 加载中 | 空状态 | 错误 | 成功 | 部分数据 |
|------|--------|--------|------|------|----------|
| Dashboard 概览 | 骨架屏(4个灰色卡片) | 引导式 onboarding：①安装CLI ②运行命令 ③查看数据，带可复制CLI命令+进度指示器 | 红色横幅"数据加载失败，点击重试" | 4个数字卡片+趋势箭头 | 缺失项显示"—" |
| 热力图 | 灰色方格骨架 | 全灰方格+"你的第一个绿格子在等你" | 跟随Dashboard整体错误 | 绿色渐变+悬浮详情 | 有数据着色/无数据灰色 |
| 模型分布图 | 旋转加载圆 | "使用更多模型来解锁分布洞察" | 跟随Dashboard整体错误 | 饼图/条形图 | 1个模型=100%+提示 |
| 分享卡片 | 进度条"生成中..." | N/A(依赖Dashboard) | "卡片生成失败，请重试" | 预览+复制链接+分享按钮 | N/A |
| Profile | 骨架屏 | "这位用户还没有数据" | "页面加载失败" | 完整展示 | 有统计无热力图时仍展示统计 |

空状态设计原则：每个空状态必须包含温暖感、一个主要CTA、和上下文说明。Dashboard 空状态是 onboarding 的关键触点。

### Cold Start Strategy

新用户注册后立即看到**示例 Dashboard**（用模拟数据），让用户在安装 CLI 前就能感受产品价值。安装 CLI 并上传数据后，无缝替换为真实数据。示例数据需明确标注"这是示例数据"，避免混淆。

## Share Card Templates (Design Review)

卡片不是单一模板，而是一个**模板系统**。用户选择最适合当前心情/数据的模板分享，增加分享频率和多样性。

所有卡片尺寸：1200×630（Twitter/OG 标准）。每张卡片只聚焦一个核心数据点，层级清晰。

| 模板名 | 核心数据 | 适用场景 | 视觉主体 |
|--------|----------|----------|----------|
| Token Counter | 总 token 消耗 | 炫耀使用量 | 大数字 + 7天sparkline |
| Money Saved | 节省金额 | 强调订阅价值 | 金额 + 对比计算 |
| ROI Beast | ROI 倍数 | 最有分享动力 | 倍数 + 订阅vs等价API |
| Full Stats | 综合数据 | 全面展示 | 3个数字 + sparkline + 徽章 |
| Model Mix | 模型分布 | 极客风格 | 水平进度条(各模型占比) |

每张卡片底部统一：Burnlog Logo + Profile URL（burnlog.dev/u/username，引流入口）。顶部统一：头像 + @用户名 + 时间范围。

### AI Slop 防护

以下模糊描述已替换为具体设计决策：
- ~~"精美的使用卡片"~~ → 5 种具体模板，各有明确信息层级

## Visual Direction (Design Review)

### 视觉调性

**关键词：** 数据专业感 × 社交趣味性 — 像 Linear 的精致度 + Strava 的成就感

- **不是**：企业 BI 工具（太冷）、游戏化 app（太闹）、通用 SaaS 模板（太无聊）
- **是**：开发者信任的数据工具，偶尔让你微笑的小惊喜

### 核心视觉方向（待 /design-consultation 细化为 DESIGN.md）

| 维度 | 方向 | 说明 |
|------|------|------|
| 色调 | 深色主题为主 + 亮色数据 | 开发者偏好深色；数据可视化用高饱和亮色突出 |
| 主色 | 绿色系（呼应 GitHub 贡献图的心智模型） | 热力图绿色渐变、增长趋势绿色、ROI 正值绿色 |
| 强调色 | 橙/金色 — 成就/徽章系统 | 解锁、里程碑、分享 CTA 用暖色调 |
| 字体方向 | Monospace 用于数据数字，Sans-serif 用于文本 | 数字用等宽字体增强专业感和对齐美感 |
| 间距 | 宽松 — 数据需要呼吸空间 | Dashboard 信息密度高，间距防止视觉压迫 |
| 图表风格 | 极简、无多余装饰 | 去掉网格线、减少轴标签、让数据本身说话 |
| 动画 | 克制 — 只在有意义的时刻 | 数字加载时的计数动画、徽章解锁时的微动画 |

> **实现前必须运行 `/design-consultation`** 生成完整 DESIGN.md，包含具体色值、字体选择、间距规模、组件库选择。

## Responsive & Accessibility (Design Review)

### 响应式策略

| 页面 | 策略 | 移动端布局变化 |
|------|------|---------------|
| Landing | **移动优先** | 单列布局，CTA按钮全宽，示例卡片缩放适配 |

| Share Card | **移动优先** | 卡片图片自适应宽度，CTA按钮全宽 |
| Dashboard | **桌面优先，移动可用** | 概览卡片 4列→2列→1列，图表全宽堆叠，热力图横向滚动 |
| Settings | **桌面优先** | 表单类页面，自然适配 |

断点：`sm: 640px` / `md: 768px` / `lg: 1024px` / `xl: 1280px`

### 无障碍基线 (V1)

- 语义化 HTML：`<nav>`, `<main>`, `<section>`, `<article>`，不滥用 `<div>`
- 键盘导航：所有交互元素可 Tab 到达，图表可用方向键浏览数据点
- ARIA 标签：图表/热力图提供 `aria-label` 描述数据含义
- 色彩对比：文本对比度 ≥ 4.5:1 (WCAG AA)
- 触摸目标：移动端按钮/链接最小 44×44px
- 暗色模式：主题色自适应 `prefers-color-scheme`（深色主题为默认）

## Landing Page Structure (Design Review)

以示例卡片为视觉中心，不是传统文字 hero。

### 页面结构（从上到下）

1. **顶部导航**：Logo + 登录/注册 CTA
2. **Hero 区域**：一句话价值主张（左） + 动态示例分享卡片（右，自动轮播不同模板）
   - 价值主张示例："Know exactly what your AI costs — and show it off."
   - CTA 按钮："Start Tracking (Free)"
3. **3 个价值点**（横排，移动端堆叠）：
   - 追踪：你花了多少 token，值多少钱
   - 洞察：模型偏好、使用趋势、热力图
   - 分享：一键生成精美卡片，炫耀你的 AI 使用量
4. **社交证明**：已有 N 个开发者在追踪（或早期用户推荐语）
5. **底部 CTA**：重复注册按钮 + "`npx burnlog sync` 一条命令开始"
